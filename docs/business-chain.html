Como Funciona

Do pedido ao resultado

Vou te explicar como uma tarefa viaja do momento em que você pede até eu te entregar o resultado. É como acompanhar uma encomenda pelo rastreio.

O que acontece em cada etapa

Etapa 1 — Entrada

Como minha mensagem chega até você?

Quando você me manda uma mensagem — seja pelo Telegram, Discord, terminal ou qualquer outro lugar — ela chega num lugar só: o Gateway. É como um correio que recebe cartas de todo mundo e entrega no endereço certo.

Você (pelo Telegram): "Organiza meus downloads" # O que acontece nos bastidores: # 1. Telegram recebe sua mensagem # 2. Gateway identifica que é você # 3. Gateway cria uma sessão para você # 4. Sessão envia para o agente processar

Detalhe: Se você mandar uma mensagem de voz, ela é automaticamente transcrita para texto. E se você continuar a conversa amanhã, eu lembro de tudo que falamos.

Etapa 2 — Raciocínio

Como eu "penso" na sua tarefa?

É aqui que entra a Inteligência Artificial. Eu envio sua mensagem para um modelo de IA (como GPT, Claude ou outros) e peço para ele pensar: "O que esse usuário quer? Quais ferramentas eu preciso usar?"

# O modelo de IA recebe: { "mensagem": "Organiza meus downloads", "memoria": "Na última vez, o usuário separou por: Imagens, Documentos", "ferramentas": ["ler_arquivos", "mover_arquivos", "criar_pasta"] } # O modelo responde: # "Vou usar: ler_arquivos → criar_pasta → mover_arquivos" # "Primeiro preciso saber onde está a pasta Downloads"

Detalhe: O modelo também consulta minha memória — ele sabe que na última vez você organizou de um jeito específico. E se a tarefa for complexa, ele pode usar vários modelos diferentes ao mesmo tempo!

Etapa 3 — Execução

Como eu executo as tarefas?

Aqui eu saio do papel e faço as coisas de verdade. Eu tenho mais de 40 ferramentas — cada uma é como uma "aptidão" que eu uso para resolver seu problema.

# Para organizar downloads, eu faço: # Passo 1: Ler os arquivos da pasta Downloads ler_arquivos("/Users/seu-nome/Downloads") # Passo 2: Criar pastas organizadas criar_pasta("/Users/seu-nome/Downloads/Imagens") criar_pasta("/Users/seu-nome/Downloads/Documentos") criar_pasta("/Users/seu-nome/Downloads/Videos") # Passo 3: Mover cada arquivo para a pasta certa mover_arquivo("foto.jpg" → "Imagens/") mover_arquivo("relatorio.pdf" → "Documentos/") mover_arquivo("video.mp4" → "Videos/")

Detalhe: Cada ferramenta tem uma verificação de segurança. Se algo der errado, eu paro e pergunto para você antes de continuar. E tudo que eu faço fica registrado num log.

Etapa 4 — Verificação

Como eu verifico se ficou bom?

Eu não entrego um trabalho pela metade. Depois de executar, eu verifico se tudo ficou certo. Se precisar de ajustes, eu mesmo faço — sem você precisar pedir de novo.

# Depois de organizar, eu verifico: # ✅ Todos os arquivos foram movidos? # ✅ Algum arquivo ficou perdido? # ✅ As pastas foram criadas corretamente? # Se algo deu errado, eu mesmo corrijo: ler_arquivos("/Users/seu-nome/Downloads") # → Ah, sobrou 1 arquivo. Vou mover ele também. mover_arquivo("apresentacao.pptx" → "Documentos/") # Máximo de 90 rodadas de verificação # (para não ficar em loop infinito)

Detalhe: Se a tarefa for perigosa (tipo deletar arquivos), eu pauso e pergunto: "Tem certeza?". Só continuo quando você aprovar.

Etapa 5 — Saída

O que eu entrego para você?

Não é só a resposta que eu entrego. Eu também guardo o que aprendi para nas próximas vezes fazer melhor. É como se eu tivesse um "caderno de anotações" que sempre fica atualizado.

Mangaba: "Pronto! Organizei seus downloads em 3 pastas: Imagens (12 arquivos), Documentos (8 arquivos) e Videos (3 arquivos). Ficou tudo no lugar!" # Nos bastidores, eu também: # 🧠 Salvei como "habilidade": organizar downloads # 📝 Guardei na memória: "Usuário gosta de separar por tipo" # 📊 Registrei: "Usei 3 ferramentas, 2 rodadas"

Detalhe: Na próxima vez que você pedir para organizar, eu já vou saber exatamente como você gosta — e faço mais rápido!

Exemplos Reais

Veja o fluxo funcionando

Cada tarefa que você me pede passa pelo mesmo fluxo. Veja como fica na prática:

📊
Relatório de Vendas
Você pede: "Gera um relatório das minhas vendas do mês"

O que eu faço:
1. Conecto na sua planilha
2. Leio os dados
3. Calculo os totais
4. Gero gráficos bonitos
5. Crio o relatório em PDF
6. Te mando o arquivo pronto
🔍
Auditoria de Código
Você pede: "Verifica se tem erros no meu site"

O que eu faço:
1. Baixo o código do GitHub
2. Escaneio tudo procurando erros
3. Encontro 3 vulnerabilidades
4. Corrijo cada uma
5. Crio um Pull Request
6. Te mando o link do PR
📧
Triagem de Emails
Você pede: "Organiza meus emails por urgência"

O que eu faço:
1. Leio todos os emails
2. Classifico por urgência
3. Extraio ações pendentes
4. Crio tarefas no seu projeto
5. Te mando um resumo
🤖
Deploy Automático
Você pede: "Publica a nova versão do site"

O que eu faço:
1. Rodo todos os testes
2. Faz o build da aplicação
3. Subo no servidor
4. Verifico se está funcionando
5. Te notifico no Telegram

Como funciona por baixo dos panos

Se você é programador e quer entender o código, aqui vai uma visão técnica (mas continue simples):

# Loop principal do Mangaba Agent (simplificado) while eu_nao_terminei: # 1. Enviar mensagem para o modelo de IA resposta = modelo_enviar( mensagem="O usuário pediu: organizar downloads", ferramentas=["ler", "mover", "criar"] ) # 2. Se a IA quer usar uma ferramenta if resposta.quer_usar_ferramenta: resultado = executar_ferramenta( ferramenta=resposta.ferramenta, argumentos=resposta.argumentos ) # 3. Mandar o resultado de volta para a IA contexto.append(resultado) else: # 4. A IA deu a resposta final return resposta.texto # 5. Aprendizado: extrair habilidades salvar_habilidade(o_que_aprendi) atualizar_memoria(preferencias_do_usuario)
